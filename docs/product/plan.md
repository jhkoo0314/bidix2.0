## **개요**

본 문서는 'BIDIX'라는 초기 아이디어를 구체화하고, MVP 개발의 방향성을 설정하기 위해 작성된 최초의 실행 계획서이다. 이는 사용자의 학습 몰입도와 서비스의 지속 가능성에 대한 초기 전략을 담고 있으며, 이후 **'BIDIX'** 라는 최종 브랜드 철학이 담긴 `PRDv2.1.md`로 발전하는 사상적 기반이 되었다.

## **1. 초기 프로젝트 정의**

### **1.1 BIDIX란 무엇인가 (초기 비전)**

BIDIX는 **실패가 불가능한 기존의 학습 환경**에 대한 문제의식에서 출발했다. 실제 경매와 동일한 리스크와 변수를 가진 가상의 환경을 제공하여, 학습자가 **'안전하게 실패하는 경험'**을 무한히 축적할 수 있도록 설계된 AI 기반 실전 트레이닝 플랫폼을 지향했다.

#### **브랜드 슬로건의 진화**

> **초기 슬로건 (v1.0):** **Fail Fast, Learn Faster.**
> 
> **최종 슬로건 (v2.0):** **Fail Safe, Bid Better.**
> 
> **진화 배경:**  
> 초기 'Fast'는 속도를, 'Faster'는 효율을 강조했으나, 사용자 조사 결과 **"빠른 실패"보다 "안전한 실패"가 더 큰 가치**로 인식됨을 발견했다. 경매 초보자들은 '속도'보다 '심리적 안전감'을 더 원했고, '더 빠른 학습'보다 '더 나은 입찰 실력'이라는 구체적 결과를 원했다.
> 
> 최종 슬로건은 "실패의 안전성(Safe)"과 "입찰 실력의 향상(Better)"이라는 두 가지 핵심 약속을 담아 재구성되었다.

#### **초기 핵심 가치**
> **"최대한 많이 실패하고, 더 많이 배울 수 있습니다."**

이 문장은 이후 다음과 같이 발전했다:
> **"실패는 비용이 아니라 자산입니다."** (최종 브랜드 보이스)

### **1.2 v1.x 문제점 진단 및 개선 방향**

프로젝트 초기, v1.x 프로토타입의 기술적 한계를 분석하고 v2.0의 아키텍처 방향성을 설정했다.

| v1.x 문제점         | 상세                                                    | v2.0 개선 방향 (초기 구상)                  |
| :------------------ | :------------------------------------------------------ | :------------------------------------------ |
| 계산 로직 분산      | 권리·수익·세금·입찰 계산이 파일별로 분리됨              | **Auction Engine 단일 파이프라인化 (SSOT)** |
| UI·엔진 강결합      | 화면 구조 변경이 로직까지 영향                          | UI/리포트/엔진 레이어 완전 분리             |
| 기획 없는 기능 누적 | "필요하면 추가" 방식으로 복잡도 증가                    | PRD 기반 설계 → Phase 개발 체계화           |
| MVP 스코프 초과     | 원래는 입찰 시뮬레이터였으나 "정식 서비스급"으로 확장됨 | v2.0은 MVP 범위 명확화 + 확장 기반 설계     |

## **2. 초기 시장 및 사용자 분석**

### **2.1 핵심 Pain Point**

> **"나는 경매를 배웠지만, 경매를 해본 적은 없다."**

이 문제의식은 최종적으로 다음과 같이 발전했다:

> **"우리는 경매를 '배워본 적 없는 사람'이 아니라, '해본 적 없는 사람'을 위해 존재합니다."**  
> (BIDIX 최종 브랜드 스토리 中)

이 문제를 해결하기 위해, BIDIX는 '정보 제공'이 아닌 **'가상 실전 훈련'**이라는 새로운 시장 카테고리를 창출하는 것을 목표로 삼았다.

### **2.2 초기 타겟 사용자**

| 유형                   | 특징                                  | 니즈                                 |
| :--------------------- | :------------------------------------ | :----------------------------------- |
| **경매 입문자**        | 유튜브·강의만 듣고 실전은 못해본 사람 | 안전하게 '실전 감각'부터 익히고 싶다 |
| **투자 준비자**        | 소액 경매·빌라·법원 경매 관심 ↑       | 리스크·권리·수익 판단 능력 필요      |
| **실전 경험자(1~3회)** | 한두 번 실패·낙찰 실패 경험           | **케이스별 판단력·속도 향상 필요**   |
| **강의/멘토 제공자**   | 교육자·PM·컨설턴트                    | "이론이 아닌 경험형 학습 도구" 필요  |

## **3. 초기 가치 제안 (Value Proposition)**

- **AI 기반 시나리오 생성:** 무한에 가까운 실전 시나리오 제공.
- **실패 경험의 자산화:** 데이터 기반의 **'복기(復碁) 리포트'**를 통한 학습.
- **체계적인 성장 경험:** `챌린지 모드` 등을 통한 목표 지향적 학습.
- **몰입도 높은 훈련 환경:** `가상 자본금`, `횟수 제한` 등을 통한 신중한 의사결정 유도.

### **3.5 v1.0 → v2.0 가치 제안 진화**

초기에 개별적으로 나열되던 가치들이, 최종적으로 **'ix' 가치 사슬**이라는 하나의 통합된 스토리로 발전했다.

| v1.0 (초기 구상) | v2.0 (최종 확정) | 설명 |
|:---|:---|:---|
| AI 기반 시나리오 생성 | **Infinite eXperience** (무한한 경험) | 단순 "많은 시나리오"에서 → "무한한 실전 경험 축적"이라는 가치로 승격 |
| 실패 경험의 자산화 | **Insight** (날카로운 통찰) | "복기 리포트"에서 → "데이터 기반 통찰 획득"이라는 학습 과정으로 명확화 |
| 체계적인 성장 경험 | **Index** (성장의 지표) | "레벨/포인트"에서 → "정확성/수익성/안정성 3가지 점수로 실력 증명"이라는 구체적 지표로 발전 |

**핵심 변화:**  
초기에는 "기능 나열" 수준이었으나, 최종적으로는 **Experience → Insight → Index**라는 **사용자의 성장 여정(User Journey)**을 명확히 보여주는 가치 사슬로 재구성되었다.

- **설계 철학:** SSOT, Pure Function, 레이어 분리 원칙 준수.

## **5. 초기 MVP 정의 (v2.0 기준)**

> **[참고]** 아래 MVP 범위는 초기 구상 단계의 내용이며, **최종 확정된 MVP 범위는 `PRDv2.1.md`를 따릅니다.**

### **5.1 포함 기능 (초기 구상)**

- AI 매물 생성 (일일 횟수 제한)
- 입찰 UI → 결과 팝업
- 권리/수익 요약 정보
- 게임 대시보드 + 포인트 시스템
- **'일일 챌린지 3종'** (이후 Post-MVP로 연기됨)
- 상세 리포트 (UI 상 잠금 표시)

### **5.2 제외 기능 (초기 구상)**

- 실매물 조회 및 연동
- 사용자 간 경쟁 입찰
- 모바일 네이티브 앱
- 가상 자본금 시스템 (이후 Post-MVP로 연기됨)

## **7. 브랜드 아이덴티티 발전 과정 (v1.0 → v2.1)**

### **7.1 초기 vs 최종 메시지 비교**

| 구분 | v1.0 (초기) | v2.1 (최종) |
|:---|:---|:---|
| **슬로건** | Fail Fast, Learn Faster | **Fail Safe, Bid Better** |
| **핵심 가치** | "많이 실패하고, 더 많이 배운다" | **"안전하게 실패하고, 더 나은 입찰자가 된다"** |
| **타겟 정의** | "경매를 배웠지만 해본 적 없는 사람" | **"배워본 적 없는 사람이 아니라, 해본 적 없는 사람"** |
| **가치 구조** | 개별 기능 나열 | **'ix' 가치 사슬 (Experience→Insight→Index)** |
| **톤** | 기능 중심 설명 | **사용자 공감 중심 스토리텔링** |

### **7.2 주요 변화의 배경**

1. **"Fast" → "Safe"로의 전환**
   - 초기 가설: "빠른 반복 학습이 핵심이다"
   - 실제 발견: "심리적 안전감이 학습 몰입의 전제조건이다"
   
2. **"Learn Faster" → "Bid Better"로의 전환**
   - 초기 가설: "학습 속도가 핵심 차별화 요소다"
   - 실제 발견: "사용자는 '학습'보다 '실력 향상'이라는 구체적 결과를 원한다"

3. **기능 나열 → 가치 사슬 구조화**
   - 초기: "이런 기능들이 있습니다"
   - 최종: "이런 여정을 통해 성장합니다 (Experience → Insight → Index)"

## **8. 아카이브 문서로서의 가치**

이 문서는 다음과 같은 이유로 보존된다:

1. **사상의 원류:** BIDIX의 핵심 문제의식("실전 경험 부재")은 초기부터 일관되게 유지되었음을 보여준다.
2. **진화의 기록:** 초기 아이디어가 어떤 과정을 거쳐 최종 브랜드로 발전했는지 추적 가능하다.
3. **의사결정의 맥락:** 왜 특정 기능이 제외되거나 연기되었는지, 그 이유를 이해할 수 있다.
4. **미래 참조:** 향후 v2.1, v3.0 개발 시 초기 비전으로 돌아가 방향성을 재확인할 수 있다.
